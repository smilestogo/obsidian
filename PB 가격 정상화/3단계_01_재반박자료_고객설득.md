# 3단계 — 고객 반박에 대한 재반박 및 협의 제안 (개정판)

> **본 자료의 성격**
> 2단계에서 정리한 고객 측 반박 논리에 대한 PB사(KOSCOM)의 재반박 자료입니다.
> 단순 방어가 아니라 **타당한 반박은 인정하고 구체적 해결책을 제시**하는 방향으로 구성되었습니다.
> 본 개정판의 핵심 메시지: **"이번 가격 개편은 운영비 정상화이며, 그 대가로 PB사가 신규 개발 부담을 단독으로 떠안고 대형사 수준 기능을 평준화한다."**

## 0. 기본 입장 — 인정할 것과 견지할 것

### 우리가 인정하는 것 (Concessions)

| 고객 반박                    | 우리의 인정                                                                                     |
| ------------------------ | ------------------------------------------------------------------------------------------ |
| "19년 동결은 PB사의 자발적 영업 전략" | ✓ **사실입니다.** BASE21→PowerBASE 마이그레이션 유도를 위한 의도적 선택이었습니다. 다만 그 누적 효과가 임계점에 도달했음을 양해 부탁드립니다. |

| "이관사 비교는 PB 내부 문제"       | ✓ **부분 인정.** 고객 부담의 직접 근거로는 부적절했음을 인정합니다. 다만 ASP 모델의 효율성 증거로 활용한 부분만 견지합니다.                |

| "소형사 형평성 미흡"             | ✓ **인정.** 매출 규모별 차등 적용 모델을 신규 제안합니다.                                                       |
| "TCO 1,355억 비교의 과장성"     | ✓ **부분 인정.** PB 동등 범위만 분리한 정밀 추정으로 재제시합니다.                                                 |

### 우리가 견지하는 것 (Non-negotiables)

1. **가격 개편의 필요성 자체**는 협의 대상이 아닙니다.
2. **'27년 TR요금 시행, '29년 기본이용료 시행**의 시점은 견지합니다 (단, 시행 방식은 협의 가능).
3. **장기적 사업 지속을 위한 인상 누적 폭**은 양보할 수 없는 영역입니다 (단, 분산·완화는 협의 가능).

> **재반박의 기본 원칙:**
> "인상의 필요성"은 협의 대상이 아니지만, "인상의 방식·속도·완화 장치 + 신규 가치 제공"은 모두 협의 가능합니다.

## I. 자동인상 조항에 대한 재반박 — 구체적 안전장치 제안

### 고객 우려의 정당성

고객사의 우려는 **전적으로 타당합니다**. 산정식·상한·협의권 없는 자동인상 조항은 어떤 CFO도 받아들일 수 없습니다. 이 부분에 대한 우리의 표현 부족을 인정하며, 다음과 같은 **구체적 안전장치**를 제안합니다.

### 제안하는 자동인상 안전장치 (4중 보호)

| 보호 장치 | 내용 |
|---|---|
| **① 산정식 명시** | 인상률 = max(0, 한국 CPI - 1.0%) + α (α는 IT 인프라 비용 지수 반영, 매년 별도 산출·공개) |
| **② 인상률 상한 (cap)** | **연 최대 +4%** (글로벌 ASP 표준 5~7% 대비 보수적) |
| **③ 누적 상한** | **5년 누적 최대 +18%** 초과 시 자동인상 중단 |
| **④ 고객 협의권** | 매년 인상 적용 90일 전 사전 통보, 30일간 협의 기간, 합의 미달 시 **외부 중재 절차** |

### 차등 인상 모델의 효과

| Tier | 적용 대상 | TR요금 시행 | 기본이용료 시행 | 자동인상 cap | 5년 누적 부담 |
|---|---|---|---|---|---|
| **Tier 1 (대형)** | PB매출 50억+ | '27년 즉시 | '29년 즉시 | 연 4% | 100% |
| **Tier 2 (중형)** | 20~50억 | '27년 즉시 | '29~'30년 분산 | 연 3% | 약 75% |
| **Tier 3 (소형)** | 20억 미만 | '27~'29년 3년 분산 | '30~'32년 3년 분산 | 연 2% | 약 50% |

> **재반박 요지:**
> - 인상의 **필요성과 시점은 견지**하되, **속도와 폭은 매출 규모에 따라 차등** 적용합니다.
> - 소형사의 매출 대비 부담률 증가를 **+0.2%p 이내**로 제한합니다.
> - PB사 전체 매출 증가는 약 -10% 감소하나, **고객사 형평성 확보가 더 중요**하다고 판단합니다.

## III. TCO 비교에 대한 재반박 — 정밀화된 데이터

### 고객 반박의 정당성

"이관사 평균 211억은 PB 외 영역(자체 영업시스템, MTS, HTS, 분석시스템 등) 모두 포함이며, PB 동등 범위만이라면 30~50억 수준"이라는 지적은 **정확합니다**. 1단계 보강자료의 1,355억 비교는 과장이었음을 인정합니다.

### 정밀화된 TCO 재추정

**이관사 전산운용비 분해 분석 (금감원 공시 + 실무 추정)**

| 비용 항목 | 이관사 평균 비중 | 5년 누적 (211억/년 × 5) |
|---|---:|---:|
| PB 동등 영역 (원장·계좌·체결) | 약 25% | **약 264억** |
| MTS·HTS·웹 (PB 외 영역) | 약 30% | 약 317억 |
| 자체 영업시스템·CRM | 약 20% | 약 211억 |
| 분석·리스크·컴플라이언스 | 약 15% | 약 158억 |
| 통신·보안·기타 | 약 10% | 약 105억 |
| **합계** | **100%** | **1,055억** |

→ **PB 동등 범위만 자체구축 시 5년 TCO ≈ 264억 + 초기 CapEx 80억 = 약 344억**

### 고객 반박의 정당성

"가격 인상의 정당성을 확보하려면 고객 가치의 정량적 향상이 필요"하다는 지적에 **전적으로 동의**합니다. 1·2단계 자료에서 가장 약했던 부분이며, 본 재반박 자료에서 핵심 보강 영역으로 다룹니다.

본 영역은 단순 SLA 강화 차원을 넘어, **PB ASP 계약 모델 자체의 구조적 업그레이드**를 제안합니다.

## VII. 종합 — "협상이 아닌 협력으로의 전환"

### 1·2단계와 3단계의 차이

| | 1단계 (당초안) | 3단계 (수정안) |
|---|---|---|
| **자동인상** | "매년 자동인상" 모호 | **연 4% cap + 누적 18% cap + 4중 안전장치** |
| **형평성** | 일률 적용 | **3-Tier 차등 인상 모델** |
| **TCO 비교** | 1,355억 (과장) | **344억 (정밀 재추정)** |
| **계약 모델** | 분담형 (모든 비용 고객 분담) | **★ 분리형 (신규 개발 PB사 단독 부담)** |
| **신규 가치** | 모호한 약속 | **★ 대형사 평준화 5년 로드맵 (450~650억 PB 부담)** |
| **시스템 안정성** | 변경 없음 | **SLA 99.99% + 자동 환급** |
| **일정** | 5개월 일방 통보 | **3개월 협의 + 단계 인상** |
| **소형사** | 반발 예상만 | **50% 인상 + 3년 분산** |
| **거버넌스** | 통보 방식 | **분기별 고객 협의체** |

### 우리가 제안하는 합의 패키지 (개정)

1. **가격 인상의 필요성**은 양측 합의 → '27년 TR요금, '29년 기본이용료 시행 견지
2. **인상의 방식**은 차등 + 분산 + cap 적용 → 형평성·예측가능성 확보
3. **★ ASP 계약 모델은 분리형 전환** → 신규 개발은 PB사 단독 부담, 운영비만 분담
4. **★ 대형사 대비 기능 평준화 5년 로드맵** → 8개 영역 약 450~650억 PB사 단독 투자
5. **시스템 안정성**은 SLA 99.99% 강제 + 환급 조항으로 정량화
6. **거버넌스**는 분기별 협의체로 전환 → 협력 파트너 관계 정립
7. **5년 안정 계약** 체결 → 양측 모두에게 예측 가능성 부여
